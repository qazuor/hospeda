# Internationalization (i18n) Guide

Complete guide to implementing multi-language support in Hospeda Web App.

---

## 📖 Overview

Hospeda Web App supports **Spanish (es)** and **English (en)** using a custom i18n package (`@repo/i18n`).

**Key Features**:

- Type-safe translations
- Lazy-loaded translation files
- Framework-agnostic (works with Astro and React)
- URL-based language selection
- Fallback to default language
- Parameter interpolation

**Supported Languages**:

- 🇦🇷 Spanish (`es`) - Default
- 🇺🇸 English (`en`)

## 🔄 Language Detection & Switching

### URL-Based Language Detection

```astro
---
// src/pages/[lang]/index.astro
import type { GetStaticPaths } from 'astro';
import { setLocale } from '../../i18n';

export const getStaticPaths = (() => {
  return [
    { params: { lang: 'es' } },
    { params: { lang: 'en' } }
  ];
}) satisfies GetStaticPaths;

const { lang } = Astro.params;

// Set locale based on URL
setLocale(lang as 'es' | 'en');
---

<!-- Page content with translations -->
```text

**URLs**:

- `/es/` - Spanish homepage
- `/en/` - English homepage
- `/es/alojamientos` - Spanish accommodations
- `/en/accommodations` - English accommodations

### Middleware-Based Detection

```ts
// src/middleware.ts
import { defineMiddleware } from 'astro:middleware';
import { setLocale, getLocale } from './i18n';

export const onRequest = defineMiddleware(async (context, next) => {
  // Get language from URL
  const urlLang = context.url.pathname.split('/')[1];

  if (urlLang === 'es' || urlLang === 'en') {
    setLocale(urlLang);
  } else {
    // Detect from browser
    const acceptLang = context.request.headers.get('accept-language');
    const browserLang = acceptLang?.split(',')[0]?.split('-')[0];

    if (browserLang === 'en') {
      setLocale('en');
    } else {
      setLocale('es'); // Default to Spanish
    }
  }

  return next();
});
```markdown

### Redirect to Preferred Language

```astro
---
// src/pages/index.astro
import { getLocale } from '../i18n';

const preferredLang = getLocale();

// Redirect to language-specific page
return Astro.redirect(`/${preferredLang}/`);
---
```markdown

### Persist Language Choice

```tsx
// src/components/ui/LanguageSwitcher.tsx
import { useStore } from '@nanostores/react';
import { locale, setLocale } from '../../i18n';
import { useEffect } from 'react';

export function LanguageSwitcher() {
  const currentLocale = useStore(locale);

  // Load saved preference
  useEffect(() => {
    const saved = localStorage.getItem('preferred-language');
    if (saved === 'es' || saved === 'en') {
      setLocale(saved);
    }
  }, []);

  // Save preference on change
  const handleChange = (newLocale: 'es' | 'en') => {
    setLocale(newLocale);
    localStorage.setItem('preferred-language', newLocale);

    // Redirect to language-specific URL
    const currentPath = window.location.pathname;
    const newPath = currentPath.replace(/^\/(es|en)/, `/${newLocale}`);
    window.location.href = newPath;
  };

  return (
    <select value={currentLocale} onChange={(e) => handleChange(e.target.value as 'es' | 'en')}>
      <option value="es">🇦🇷 Español</option>
      <option value="en">🇺🇸 English</option>
    </select>
  );
}
```text

### Pluralization

```json
{
  "accommodation": {
    "count": {
      "zero": "No hay alojamientos",
      "one": "1 alojamiento",
      "other": "{count} alojamientos"
    }
  }
}
```text

**Custom Helper**:

```ts
// src/i18n/helpers.ts
import { t } from './index';

export function pluralize(key: string, count: number): string {
  if (count === 0) return t(`${key}.zero`, { count });
  if (count === 1) return t(`${key}.one`, { count });
  return t(`${key}.other`, { count });
}
```text

**Usage**:

```tsx
import { pluralize } from '../../i18n/helpers';

pluralize('accommodation.count', 0)   // "No hay alojamientos"
pluralize('accommodation.count', 1)   // "1 alojamiento"
pluralize('accommodation.count', 5)   // "5 alojamientos"
```text

## 🔗 URL Structure

### Path-Based Language URLs

```text
Recommended structure:

/es/                     → Spanish homepage
/es/alojamientos         → Spanish accommodations
/es/alojamientos/hotel-1 → Spanish accommodation detail

/en/                     → English homepage
/en/accommodations       → English accommodations
/en/accommodations/hotel-1 → English accommodation detail
```markdown

### Dynamic Routes

```astro
---
// src/pages/[lang]/alojamientos/[slug].astro
import type { GetStaticPaths } from 'astro';
import { setLocale } from '../../../i18n';
import { AccommodationService } from '@repo/service-core';

export const getStaticPaths = (async () => {
  const service = new AccommodationService();
  const result = await service.findAll();

  if (!result.success) return [];

  const paths = [];

  // Generate for each language
  for (const lang of ['es', 'en']) {
    for (const acc of result.data.items) {
      paths.push({
        params: {
          lang,
          slug: acc.slug
        },
        props: {
          accommodation: acc,
          lang
        }
      });
    }
  }

  return paths;
}) satisfies GetStaticPaths;

const { accommodation, lang } = Astro.props;
setLocale(lang);
---

<!-- Page content with i18n -->
```markdown

## ✅ Best Practices

### 1. Use Namespaces

```json
// ✅ Good: Organized by feature
{
  "auth": { ... },
  "accommodation": { ... },
  "booking": { ... }
}

// ❌ Bad: Flat structure
{
  "loginButton": "...",
  "accommodationTitle": "...",
  "bookingConfirm": "..."
}
```markdown

### 2. Keep Keys Descriptive

```json
// ✅ Good: Clear intent
{
  "accommodation": {
    "bookNowButton": "Reservar ahora",
    "pricePerNightLabel": "por noche"
  }
}

// ❌ Bad: Generic keys
{
  "btn1": "Reservar ahora",
  "label2": "por noche"
}
```markdown

### 3. Don't Hardcode Text

```tsx
// ❌ Bad: Hardcoded text
<h1>Bienvenido a Hospeda</h1>

// ✅ Good: Use translations
<h1>{t('home.welcome')}</h1>
```markdown

### 4. Provide Context in Parameters

```json
// ✅ Good: Clear parameter names
"greeting": "Hola, {userName}. Tienes {messageCount} mensajes."

// ❌ Bad: Generic parameter names
"greeting": "Hola, {name}. Tienes {count} mensajes."
```markdown

### 5. Handle Missing Translations

```ts
// src/i18n/index.ts
export const { t } = createI18n({
  defaultLocale: 'es',
  locales: ['es', 'en'],
  translations: {
    es: () => import('./translations/es.json'),
    en: () => import('./translations/en.json')
  },
  onMissingTranslation: (key, locale) => {
    console.warn(`Missing translation: ${key} for ${locale}`);
    return key; // Return key as fallback
  }
});
```text

## 🚫 Common Mistakes

### Mistake 1: Mixing Languages in Code

```tsx
// ❌ Wrong: Hardcoded Spanish
<button>Buscar</button>

// ✅ Correct: Use i18n
<button>{t('common.search')}</button>
```markdown

### Mistake 2: Not Using Parameters

```json
// ❌ Bad: Separate keys for each variation
{
  "welcome1": "Bienvenido",
  "welcome2": "Bienvenido, Juan",
  "welcome3": "Bienvenido, María"
}

// ✅ Good: Single key with parameter
{
  "welcome": "Bienvenido{name, select, null {} other {, {name}}}"
}
```text

Or simpler:

```json
{
  "welcomeGeneric": "Bienvenido",
  "welcomePersonal": "Bienvenido, {name}"
}
```markdown

### Mistake 3: Direct Date/Number Formatting

```tsx
// ❌ Bad: Hardcoded format
const formatted = `${day}/${month}/${year}`;

// ✅ Good: Use locale-aware formatter
const formatted = new Intl.DateTimeFormat(currentLocale).format(date);
```markdown

### Mistake 4: Not Lazy Loading Translations

```ts
// ❌ Bad: Import translations directly
import esTranslations from './translations/es.json';
import enTranslations from './translations/en.json';

// ✅ Good: Lazy load
translations: {
  es: () => import('./translations/es.json'),
  en: () => import('./translations/en.json')
}
```text

## 🧪 Testing i18n

### Test Translations

```ts
// test/i18n/translations.test.ts
import { describe, it, expect } from 'vitest';
import { t, setLocale } from '../../src/i18n';

describe('i18n translations', () => {
  it('should translate to Spanish', () => {
    setLocale('es');
    expect(t('common.welcome')).toBe('Bienvenido');
  });

  it('should translate to English', () => {
    setLocale('en');
    expect(t('common.welcome')).toBe('Welcome');
  });

  it('should interpolate parameters', () => {
    setLocale('es');
    expect(t('greeting', { name: 'Juan' })).toBe('Hola, Juan');
  });

  it('should fallback to default locale', () => {
    setLocale('es');
    const result = t('nonexistent.key');
    expect(result).toBe('nonexistent.key');
  });
});
```markdown
