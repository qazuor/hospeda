## Understanding Dependencies

### What are Seed Dependencies?

Dependencies occur when one entity references another via foreign keys:

```typescript
// Accommodation depends on User and Destination
{
  "id": "accommodation-001",
  "name": "Beach Hotel",
  "ownerId": "user-001",           // Foreign key → users
  "destinationId": "destination-001" // Foreign key → destinations
}
```

**Rule:** Parent entities must be seeded **before** children.

### Dependency Types

#### 1. One-to-Many (Foreign Key)

```typescript
// Post belongs to one User
{
  "id": "post-001",
  "title": "Travel Guide",
  "authorId": "user-001"  // FK → users (one-to-many)
}
```

#### 2. Many-to-Many (Junction Table)

```typescript
// Accommodation has many Amenities through junction table
{
  "id": "accommodation-001",
  "name": "Beach Hotel",
  "amenityIds": ["amenity-001", "amenity-002"]  // M2M → amenities
}
```

#### 3. Self-Referencing

```typescript
// Comment can reply to another Comment
{
  "id": "comment-002",
  "content": "Great point!",
  "parentCommentId": "comment-001"  // FK → comments (self-ref)
}
```

## Execution Order

### Required Seeds Order

```typescript
// packages/seed/src/required/index.ts
export async function runRequiredSeeds(context: SeedContext): Promise<void> {
  // No dependencies
  await seedRolePermissions(context);  // 1. Foundation data

  // No dependencies
  await seedAmenities(context);        // 2. Amenities
  await seedFeatures(context);         // 3. Features

  // No dependencies
  await seedAttractions(context);      // 4. Attractions

  // Depends on: Attractions
  await seedDestinations(context);     // 5. Destinations (links to Attractions)
}
```

**Key principles:**

- Independent entities first
- Parent before child
- Foundation before specialized

### Example Seeds Order

```typescript
// packages/seed/src/example/index.ts
export async function runExampleSeeds(context: SeedContext): Promise<void> {
  // No dependencies
  await seedUsers(context);             // 1. Users
  await seedTags(context);              // 2. Tags
  await seedPostSponsors(context);      // 3. Post Sponsors
  await seedEventOrganizers(context);   // 4. Event Organizers
  await seedEventLocations(context);    // 5. Event Locations

  // Depends on: Users, Destinations, Amenities, Features
  await seedAccommodations(context);    // 6. Accommodations

  // Depends on: Users, Post Sponsors, Tags
  await seedPosts(context);             // 7. Posts

  // Depends on: Users, Event Organizers, Event Locations, Tags
  await seedEvents(context);            // 8. Events

  // Depends on: Users, Accommodations
  await seedAccommodationReviews(context);  // 9. Reviews

  // Depends on: Users, Destinations
  await seedDestinationReviews(context);    // 10. Destination Reviews

  // Depends on: Users, Accommodations, Posts, Events
  await seedBookmarks(context);         // 11. Bookmarks

  // Many-to-many relationships
  await seedPostSponsorships(context);  // 12. Post-Sponsor links
  await seedTagRelations(context);      // 13. Tag relations
}
```

```
    └─> Bookmarks

    └─> Bookmarks
```

## ID Mapping System

### How It Works

The ID mapper tracks relationships between:

- **Seed IDs**: Identifiers in JSON files (`user-001`, `amenity-wifi`)
- **Real IDs**: UUID generated by database (`a1b2c3d4-...`)

```typescript
// After creating entity
context.idMapper.setMapping(
  'users',              // Entity type
  'user-001',           // Seed ID from JSON
  'a1b2c3d4-uuid',      // Real UUID from database
  'John Doe'            // Optional display name
);

// Later, when creating dependent entity
const realUserId = context.idMapper.getMappedUserId('user-001');
// Returns: 'a1b2c3d4-uuid'
```

### Stored Mappings

Mappings are persisted to `mappings/id-mappings.json`:

```json
{
  "users": {
    "user-001": {
      "id": "a1b2c3d4-e5f6-4789-0123-456789abcdef",
      "name": "John Doe"
    }
  },
  "amenities": {
    "amenity-wifi": {
      "id": "b2c3d4e5-f6a7-4890-1234-567890abcdef",
      "name": "WiFi"
    }
  }
}
```

**Benefits:**

- Incremental seeding (don't need full reset)
- Clear error messages with names
- Debugging (inspect mappings file)
- Reproducible relationships

### Using ID Mapper

#### Store Mapping (Automatic)

Seed factory automatically stores mappings after entity creation:

```typescript
// Happens automatically in seedFactory
const result = await service.create(data);
context.idMapper.setMapping(
  entityName,
  seedId,
  result.data.id,
  entityName  // Or custom name
);
```

#### Retrieve Single ID

```typescript
// Specific getter
const userId = context.idMapper.getMappedUserId('user-001');

// Generic getter
const userId = context.idMapper.getRealId('users', 'user-001');
```

#### Retrieve Multiple IDs

```typescript
// Get all IDs (may contain undefined)
const amenityIds = context.idMapper.getRealIds('amenities', [
  'amenity-wifi',
  'amenity-parking',
  'amenity-invalid'  // Returns undefined
]);
// Result: ['uuid1', 'uuid2', undefined]

// Get only valid IDs (filters undefined)
const validAmenityIds = context.idMapper.getValidRealIds('amenities', [
  'amenity-wifi',
  'amenity-parking',
  'amenity-invalid'  // Filtered out
]);
// Result: ['uuid1', 'uuid2']
```

#### Validate Mappings

```typescript
const validation = context.idMapper.validateMappings('amenities', [
  'amenity-wifi',
  'amenity-parking',
  'amenity-invalid'
]);

if (!validation.isValid) {
  console.error('Missing amenity IDs:', validation.missingIds);
  // ['amenity-invalid']

  console.log('Valid amenity IDs:', validation.validIds);
  // ['uuid1', 'uuid2']
}
```

#### Display Names

```typescript
// Get readable name for seed ID
const displayName = context.idMapper.getDisplayName('users', 'user-001');
// Returns: 'John Doe' (or 'user-001' if no name)

// Reverse lookup: get name from real ID
const displayName = context.idMapper.getDisplayNameByRealId(
  'users',
  'a1b2c3d4-uuid'
);
// Returns: 'John Doe'
```

## Foreign Key Dependencies

### Mapping in preProcess

Handle foreign keys before entity creation:

```typescript
export const seedAccommodations = createSeedFactory({
  entityName: 'Accommodations',
  serviceClass: AccommodationService,
  folder: 'src/data/accommodation',
  files: exampleManifest.accommodations,

  preProcess: async (item, context) => {
    const accommodation = item as {
      ownerId: string;
      destinationId: string;
    };

    // Map owner ID
    const realOwnerId = context.idMapper.getMappedUserId(
      accommodation.ownerId
    );

    if (!realOwnerId) {
      throw new Error(
        `Owner not found: ${accommodation.ownerId}. ` +
        `Ensure users are seeded before accommodations.`
      );
    }

    accommodation.ownerId = realOwnerId;

    // Map destination ID
    const realDestinationId = context.idMapper.getMappedDestinationId(
      accommodation.destinationId
    );

    if (!realDestinationId) {
      throw new Error(
        `Destination not found: ${accommodation.destinationId}. ` +
        `Ensure destinations are seeded before accommodations.`
      );
    }

    accommodation.destinationId = realDestinationId;
  }
});
```

### JSON Structure

```json
{
  "$schema": "../../schemas/accommodation.schema.json",
  "id": "accommodation-beach-hotel-001",
  "name": "Beach Hotel Paradise",
  "type": "HOTEL",
  "ownerId": "user-owner-maria",      // Seed ID, will be mapped
  "destinationId": "destination-concepcion"  // Seed ID, will be mapped
}
```

### Error Messages

Good error messages include context:

```typescript
if (!realOwnerId) {
  const ownerName = context.idMapper.getDisplayName('users', seedOwnerId);
  throw new Error(
    `Owner "${ownerName}" (${seedOwnerId}) not found for ` +
    `accommodation "${accommodation.name}". ` +
    `Ensure users are seeded first.`
  );
}
```

**Output:**

```
Error: Owner "Maria Garcia" (user-owner-maria) not found for
accommodation "Beach Hotel Paradise". Ensure users are seeded first.
```

## Many-to-Many Relationships

### Using relationBuilder

Handle many-to-many relationships after entity creation:

```typescript
export const seedAccommodations = createSeedFactory({
  entityName: 'Accommodations',
  serviceClass: AccommodationService,
  folder: 'src/data/accommodation',
  files: exampleManifest.accommodations,

  normalizer: (data) => {
    // Exclude relationship IDs from main data
    const { $schema, id, slug, amenityIds, featureIds, ...cleanData } = data;
    return cleanData;
  },

  relationBuilder: async (result, item, context) => {
    const accommodationId = result.data?.id;
    if (!accommodationId) {
      throw new Error('No accommodation ID in result');
    }

    const accommodation = item as {
      amenityIds?: string[];
      featureIds?: string[];
    };

    const service = new AccommodationService({});

    // Add amenities
    const seedAmenityIds = accommodation.amenityIds || [];
    const validation = context.idMapper.validateMappings(
      'amenities',
      seedAmenityIds
    );

    if (!validation.isValid) {
      logger.warn(
        `Missing amenity mappings: ${validation.missingIds.join(', ')}`
      );
    }

    for (const amenityId of validation.validIds) {
      await service.addAmenityToAccommodation(context.actor!, {
        accommodationId,
        amenityId
      });
    }

    // Add features
    const seedFeatureIds = accommodation.featureIds || [];
    const featureIds = context.idMapper.getValidRealIds(
      'features',
      seedFeatureIds
    );

    for (const featureId of featureIds) {
      await service.addFeatureToAccommodation(context.actor!, {
        accommodationId,
        featureId
      });
    }
  }
});
```

### Using Service Relation Builder

Cleaner approach with utility:

```typescript
import { createServiceRelationBuilder } from '../utils/index.js';

const amenitiesBuilder = createServiceRelationBuilder({
  serviceClass: AmenityService,
  methodName: 'addAmenityToAccommodation',
  extractIds: (item) => (item as { amenityIds?: string[] }).amenityIds || [],
  entityType: 'amenities',
  relationType: 'amenities',
  buildParams: (accommodationId, amenityId) => ({
    accommodationId,
    amenityId
  }),
  getMainEntityInfo: (item) => {
    const acc = item as { name: string };
    return `"${acc.name}"`;
  },
  getRelatedEntityInfo: (seedId, context) => {
    return context.idMapper.getDisplayName('amenities', seedId);
  }
});

export const seedAccommodations = createSeedFactory({
  // ... other config
  relationBuilder: amenitiesBuilder
});
```

## Circular Dependencies

### Problem

Two entities reference each other:

```typescript
// User can have a favoritePost
{
  "id": "user-001",
  "name": "John Doe",
  "favoritePostId": "post-001"  // Depends on Post
}

// Post has an author
{
  "id": "post-001",
  "title": "My Post",
  "authorId": "user-001"  // Depends on User
}
```

**Issue:** Which to seed first?

### Solution 1: Nullable Foreign Key

Make one FK optional:

```typescript
// 1. Seed Users without favoritePostId
{
  "id": "user-001",
  "name": "John Doe"
  // No favoritePostId yet
}

// 2. Seed Posts with authorId
{
  "id": "post-001",
  "title": "My Post",
  "authorId": "user-001"  // Now exists
}

// 3. Update Users with favoritePostId (postProcess)
export const seedUsers = createSeedFactory({
  // ... config
  postProcess: async (result, item, context) => {
    const user = item as { favoritePostId?: string };
    if (user.favoritePostId) {
      const postId = context.idMapper.getMappedPostId(user.favoritePostId);
      if (postId) {
        await userService.update(result.data.id, {
          favoritePostId: postId
        });
      }
    }
  }
});
```

### Solution 2: Separate Relationship Seed

Create junction table seed after both entities exist:

```typescript
// 1. Seed Users
await seedUsers(context);

// 2. Seed Posts
await seedPosts(context);

// 3. Seed User-Post Favorites (separate seed)
await seedUserFavorites(context);
```

### Solution 3: Two-Pass Seeding

Seed entity twice:

```typescript
// Pass 1: Create entity without circular FK
export const seedUsers = createSeedFactory({
  normalizer: (data) => {
    const { favoritePostId, ...cleanData } = data;  // Exclude
    return cleanData;
  }
});

// Pass 2: Update with circular FK
export async function seedUserFavorites(context: SeedContext) {
  const files = await loadJsonFiles('src/data/user', manifest.users);

  for (const file of files) {
    if (file.favoritePostId) {
      const userId = context.idMapper.getMappedUserId(file.id);
      const postId = context.idMapper.getMappedPostId(file.favoritePostId);

      if (userId && postId) {
        await userService.update(userId, { favoritePostId: postId });
      }
    }
  }
}
```

## Validation Strategies

### Pre-Creation Validation

Validate dependencies exist before attempting creation:

```typescript
export const seedAccommodations = createSeedFactory({
  entityName: 'Accommodations',
  serviceClass: AccommodationService,
  folder: 'src/data/accommodation',
  files: exampleManifest.accommodations,

  validateBeforeCreate: async (data) => {
    const accommodation = data as {
      ownerId: string;
      destinationId: string;
    };

    // Check owner exists
    const hasOwner = context.idMapper.hasMapping('users', accommodation.ownerId);
    if (!hasOwner) {
      logger.warn(
        `Skipping accommodation: owner ${accommodation.ownerId} not found`
      );
      return false;
    }

    // Check destination exists
    const hasDestination = context.idMapper.hasMapping(
      'destinations',
      accommodation.destinationId
    );
    if (!hasDestination) {
      logger.warn(
        `Skipping accommodation: destination ${accommodation.destinationId} not found`
      );
      return false;
    }

    return true;
  }
});
```

### Batch Validation

Validate all dependencies at once:

```typescript
preProcess: async (item, context) => {
  const accommodation = item as {
    ownerId: string;
    destinationId: string;
    amenityIds: string[];
    featureIds: string[];
  };

  // Validate all foreign keys
  const checks = [
    {
      type: 'users',
      id: accommodation.ownerId,
      label: 'owner'
    },
    {
      type: 'destinations',
      id: accommodation.destinationId,
      label: 'destination'
    }
  ];

  const missing: string[] = [];

  for (const check of checks) {
    if (!context.idMapper.hasMapping(check.type, check.id)) {
      missing.push(`${check.label} (${check.id})`);
    }
  }

  if (missing.length > 0) {
    throw new Error(
      `Missing dependencies for accommodation: ${missing.join(', ')}`
    );
  }

  // Validate relationships
  const amenityValidation = context.idMapper.validateMappings(
    'amenities',
    accommodation.amenityIds
  );

  if (!amenityValidation.isValid) {
    logger.warn(
      `Missing amenities: ${amenityValidation.missingIds.join(', ')}`
    );
  }

  // Map all FKs
  accommodation.ownerId = context.idMapper.getRealId(
    'users',
    accommodation.ownerId
  )!;
  accommodation.destinationId = context.idMapper.getRealId(
    'destinations',
    accommodation.destinationId
  )!;
}
```

### Statistics Validation

Check mapping statistics before seeding:

```typescript
// Before seeding accommodations
const userStats = context.idMapper.getMappingStats('users');
if (userStats.count === 0) {
  throw new Error('No users seeded. Seed users before accommodations.');
}

const destinationStats = context.idMapper.getMappingStats('destinations');
if (destinationStats.count === 0) {
  throw new Error('No destinations seeded. Seed destinations before accommodations.');
}

logger.info(`Found ${userStats.count} users and ${destinationStats.count} destinations`);
```

## Troubleshooting

### Common Issues

#### 1. Missing ID Mapping

**Error:**

```
Error: No mapping found for owner ID: user-001
```

**Causes:**

- Parent entity not seeded yet
- Wrong seed ID in JSON
- Execution order problem

**Solutions:**

```typescript
// Check if mapping exists
if (!context.idMapper.hasMapping('users', 'user-001')) {
  // Parent not seeded
  logger.error('User user-001 not found. Ensure users are seeded first.');
}

// Print available mappings
context.idMapper.printMappingStats('users');

// Check mappings file
// mappings/id-mappings.json
```

#### 2. Foreign Key Constraint Violation

**Error:**

```
Error: insert or update on table "accommodations" violates
foreign key constraint "accommodations_owner_id_fkey"
```

**Causes:**

- Real ID not found in database
- ID mapper has stale data
- Parent entity was deleted

**Solutions:**

```typescript
// Validate before insert
const realOwnerId = context.idMapper.getMappedUserId(seedOwnerId);
if (!realOwnerId) {
  throw new Error('Owner mapping not found');
}

// Verify in database
const ownerExists = await db.query.users.findFirst({
  where: eq(users.id, realOwnerId)
});

if (!ownerExists) {
  throw new Error('Owner not in database');
}
```

#### 3. Circular Dependency

**Error:**

```
Error: Cannot create User without Post, cannot create Post without User
```

**Solutions:**

See [Circular Dependencies](#circular-dependencies) section above.

#### 4. Execution Order

**Error:**

```
Error: Destination not found: destination-001
```

**Cause:** Seeding accommodations before destinations

**Solution:** Fix execution order:

```typescript
// Wrong
await seedAccommodations(context);
await seedDestinations(context);  // Should be first!

// Correct
await seedDestinations(context);
await seedAccommodations(context);
```

### Debugging Tips

#### 1. Print ID Mappings

```typescript
// All mappings
context.idMapper.printAllMappingStats();

// Specific entity
context.idMapper.printMappingStats('users');
```

#### 2. Inspect Mappings File

```bash
cat mappings/id-mappings.json | jq .users
```

#### 3. Enable Detailed Logging

```typescript
// Check what's being processed
preProcess: async (item, context) => {
  logger.debug(`Processing: ${JSON.stringify(item, null, 2)}`);
  logger.debug(`Available user IDs: ${context.idMapper.getMappingStats('users').count}`);

  // ... rest of preProcess
}
```

#### 4. Validate Incrementally

```typescript
// After each FK mapping
const realOwnerId = context.idMapper.getMappedUserId(seedOwnerId);
console.log(`Mapped ${seedOwnerId} -> ${realOwnerId}`);

if (!realOwnerId) {
  console.log('Available users:', context.idMapper.getAllMappingStats().users);
  throw new Error('Mapping failed');
}
```

## Best Practices

1. **Document dependencies** - Comment required seeds in JSDoc
2. **Use specific getters** - `getMappedUserId()` over generic `getRealId()`
3. **Validate early** - Check mappings in `preProcess`
4. **Handle missing gracefully** - Use `getValidRealIds()` for optional relations
5. **Provide context** - Include entity names in error messages
6. **Test execution order** - Ensure seeds run in correct sequence
7. **Clear stale mappings** - Reset mappings when resetting database
8. **Log statistics** - Print mapping stats for debugging
